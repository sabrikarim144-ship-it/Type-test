🚀 Objectif QA : Zen Révision

NIVEAU 1

Unit Testing (TU)

Vérifie la plus petite unité de code. Rapide et automatique.

NIVEAU 2

Integration Testing (TI)

Vérifie si les composants "se parlent" correctement en interne.

NIVEAU 3

System Integration (SIT)

Vérifie les connexions avec les systèmes extérieurs (Paiement, SMS, etc).

NIVEAU 4

System Testing

Vérifie l'application complète. Est-ce que techniquement tout roule ?

NIVEAU 5

Acceptance (UAT)

Le client valide : "C'est exactement ce que j'ai demandé".

Type Question clé Exemple
Fonctionnel Quoi ? "L'utilisateur peut-il se connecter ?"
Non Fonctionnel Comment ? "Le site supporte-t-il 1000 connexions/sec ?"
